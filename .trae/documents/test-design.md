# AI Album 测试设计文档

## 1. 测试策略总览

### 1.1 测试目标
确保AI Album从产品定义、系统架构到功能实现的完整质量，覆盖认证授权、照片管理、AI处理链路、搜索、分享、移动端等全部功能模块，验证功能正确性、性能达标、安全合规、兼容性良好。

### 1.2 测试层级

| 层级 | 范围 | 工具 | 覆盖率目标 |
|------|------|------|-----------|
| 单元测试 | 工具函数、Zod Schema、Repository方法 | vitest + pytest | 后端≥80%, Worker≥70%, 前端≥60%, ML≥50% |
| 集成测试 | API端点、数据库交互、Worker链路 | vitest + supertest | 核心API 100%覆盖 |
| E2E测试 | 关键用户流程 | Playwright | P0功能100%覆盖 |
| 性能测试 | 响应时间、吞吐量、并发 | k6 / autocannon | 满足PRD性能指标 |
| 安全测试 | 认证、授权、注入、XSS | OWASP ZAP + 手工 | OWASP Top 10覆盖 |
| 兼容性测试 | 浏览器、设备、平台 | BrowserStack + 实机 | PRD兼容性矩阵覆盖 |

### 1.3 测试环境

| 环境 | 用途 | 配置 |
|------|------|------|
| CI环境 | 自动化测试 | GitHub Actions (linux/amd64) |
| 开发环境 | 日常开发测试 | docker-compose.dev.yml (PG+Valkey) |
| Staging | 集成/性能测试 | 完整Docker Compose (5容器) |
| 生产验证 | 发布前验证 | Ubuntu 22.04 ARM64 |

### 1.4 测试数据策略
- 标准测试图片集: 50张(含JPEG/PNG/WebP/HEIC各10张, 含EXIF/无EXIF各半, 含人脸/无脸各半, 含文字/无文字各半)
- 标准测试视频集: 10个(MP4/MOV/WebM/AVI, 含GPS/无GPS)
- 边界测试文件: 超大文件(199MB)、空文件(0KB)、损坏文件、错误MIME类型文件
- 多语言测试图片: 含中文/英文/日文文字的图片各5张
- LivePhoto测试集: Apple LivePhoto配对(JPEG+MOV)3组, Samsung MotionPhoto 2个, Google MotionPhoto 2个

## 11. ML模型精度验收标准

> ML模型精度是AI相册的核心差异化指标，必须达到以下最低标准才能发布。测试使用标准数据集，每季度重新评估。

### 11.1 OCR识别精度 (PaddleOCR)

| 指标 | 最低标准 | 目标值 | 测试数据集 | 测试方法 |
|------|----------|--------|-----------|----------|
| 中文印刷体识别率 | ≥90% | ≥95% | ICDAR 2019 MLT (中文子集) 500张 | 正确字符数/总字符数(CER) |
| 英文印刷体识别率 | ≥93% | ≥97% | ICDAR 2015 500张 | CER |
| 中英混合识别率 | ≥88% | ≥93% | 自建数据集200张(路牌/菜单/文档) | CER |
| 手写体识别率 | ≥70% | ≥80% | CASIA-HWDB 200张 | CER |
| 端到端延迟(CPU) | ≤5秒/张 | ≤3秒/张 | 1080p图片 | 计时 |
| 端到端延迟(GPU) | ≤1秒/张 | ≤0.5秒/张 | 1080p图片 | 计时 |
| 空图片误识别率 | ≤5% | ≤2% | 无文字图片100张 | 误报数/总数 |

**精度测试流程:**
1. 准备测试数据集(含ground truth文本)
2. 对每张图片执行OCR，收集识别结果
3. 计算CER(Character Error Rate) = (替换+删除+插入) / 总字符数
4. 精度 = 1 - CER
5. 不达标时: 检查PaddleOCR语言包配置→调整det/rec模型参数→更换预处理策略

### 11.2 CLIP语义搜索精度

| 指标 | 最低标准 | 目标值 | 测试数据集 | 测试方法 |
|------|----------|--------|-----------|----------|
| 中文语义搜索Recall@10 | ≥60% | ≥75% | 自建数据集1000张+50条中文查询 | 查询返回Top10中相关结果占比 |
| 英文语义搜索Recall@10 | ≥65% | ≥80% | Flickr30k 1000张+50条英文查询 | 同上 |
| 跨语言搜索(中文查英文场景) | ≥50% | ≥65% | 混合数据集500张 | 同上 |
| 智能分类准确率 | ≥85% | ≥90% | 自建数据集2000张(10类别) | 正确分类数/总数 |
| 向量索引召回率(ANN) | ≥95% | ≥99% | 10000张图片，精确搜索对比 | ANN结果与精确搜索结果重叠率 |
| 端到端延迟(CPU) | ≤10秒/张 | ≤5秒/张 | 1080p图片 | 计时(含预处理+推理) |
| 端到端延迟(GPU) | ≤2秒/张 | ≤1秒/张 | 1080p图片 | 计时 |

**精度测试流程:**
1. 准备标注数据集(每张图片有语义标签)
2. 执行50条测试查询，记录每条查询的Top10结果
3. 人工标注每个结果是否相关
4. 计算Recall@10 = 相关结果数 / 总相关结果数
5. 智能分类: 对2000张图片执行自动分类，计算分类准确率
6. 不达标时: 调整HNSW参数(ef_search/m)→调整分类阈值→更换CLIP模型

### 11.3 人脸识别精度 (InsightFace SCRFD)

| 指标 | 最低标准 | 目标值 | 测试数据集 | 测试方法 |
|------|----------|--------|-----------|----------|
| 人脸检测Recall | ≥90% | ≥95% | WIDER FACE 500张 | 检测到的人脸/总人脸 |
| 人脸检测Precision | ≥88% | ≥93% | WIDER FACE 500张 | 正确检测/总检测数 |
| 人脸聚类F1 Score | ≥85% | ≥90% | LFW 500张(50人) | 聚类正确性F1 |
| 侧脸检测率(>30°) | ≥70% | ≥80% | 自建侧脸数据集100张 | 检测率 |
| 遮挡人脸检测率 | ≥60% | ≥75% | 自建遮挡数据集100张(口罩/墨镜) | 检测率 |
| 误聚类率(不同人聚为一类) | ≤10% | ≤5% | LFW 500张 | 误聚类对数/总对数 |
| 端到端延迟(CPU) | ≤8秒/张 | ≤5秒/张 | 1080p图片(含多人脸) | 计时 |
| 端到端延迟(GPU) | ≤2秒/张 | ≤1秒/张 | 1080p图片 | 计时 |

**精度测试流程:**
1. 人脸检测: 在WIDER FACE测试集上运行SCRFD，计算Recall/Precision
2. 人脸聚类: 对LFW数据集50人执行聚类，计算F1 = 2×Precision×Recall/(Precision+Recall)
3. 侧脸/遮挡: 在专门数据集上测试极端场景
4. 不达标时: 调整检测阈值(det_thresh)→调整聚类阈值→更换模型

### 11.4 ML精度回归测试

| 触发时机 | 测试范围 | 通过标准 |
|----------|----------|----------|
| 模型版本更新 | 全部3项精度测试 | 所有指标≥最低标准 |
| 依赖库升级(PaddleOCR/InsightFace) | 对应模块精度测试 | 指标不低于上一版本 |
| 每季度常规评估 | 全部3项精度测试 | 所有指标≥最低标准 |
| 预处理流程变更 | 端到端精度+延迟测试 | 精度不降+延迟不增 |

**ML精度不达标的处理流程:**
1. 标记该ML模块为"精度不达标"，在管理页显示警告
2. 通知Admin(邮件+应用内通知)
3. 功能降级: 不达标的AI功能仍可用但搜索结果添加"精度可能降低"提示
4. 排查: 检查模型文件完整性→检查依赖版本→检查预处理参数
5. 修复后重新运行精度测试，通过后移除警告
